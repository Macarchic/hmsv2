# HMS – Методи учасників змагання

## Препроцесинг

**Перетворення в частотний домен:**
- Short-Time Fourier Transform (STFT) / спектрограми — найпоширеніший підхід, Kaggle надав готові спектрограми
- Continuous Wavelet Transform (CWT) — дає кращу часово-частотну роздільність ніж STFT, особливо для коротких сплесків
- Деякі учасники комбінували обидва представлення

**Формування сигналу:**
- Bipolar montage (Double Banana) — 19 монополярних електродів → 16 биполярних каналів; прибирає спільний шум, відповідає клінічній практиці
- Bandpass 0.5–20 Гц — відрізає drift знизу і м'язові артефакти зверху; узгоджується з діапазоном Kaggle-спектрограм
- Clip ±1024 µV + масштабування — прибирає екстремальні артефакти руху

**Формування 2D-зображення (для CNN-бекбонів):**
- Reshape часового ряду `(16, T)` → `(160, T//10)` — 16 каналів × 10 мікро-семплів по висоті, час по горизонталі
- Накладання кількох часових масштабів одного семплу як RGB-канали (наприклад 10 сек / 25 сек / 50 сек) — дає моделі одночасно дрібні деталі й загальний патерн
- Resize до фіксованого розміру (наприклад 224×224 або 160×1000) під вхід претренованого бекбону

**Мітки:**
- Усереднення голосів по всіх sub-id одного eeg_id — замість одного рядка беруть середній розподіл по всіх вікнах запису
- Label smoothing (+ε до кожного класу перед нормалізацією) — покращує калібрацію для KL-divergence метрики

---

## Аугментація

- **Left-Right flip** — міняти місцями ліву і праву півкулі (LL↔RL, LP↔RP); ЕЕГ-патерни анатомічно симетричні, клас не змінюється
- **Random temporal crop center** — зсув центру вирізки всередині 50-секундного вікна; модель не запам'ятовує позицію патерну
- **Mixup** — лінійна інтерполяція між двома семплами і їх мітками; допомагає при нечітких межах між класами
- Gaussian noise, random масштабування амплітуди

---

## Архітектура

**Бекбони (претреновані на ImageNet):**
- EfficientNet (B0–B7, V2) — найпопулярніший вибір завдяки балансу між розміром і якістю
- EfficientNetV2, ConvNeXt, ResNet, ResNeXt, RegNet
- ViT / Swin Transformer — деякі топ-рішення

**Голови класифікатора:**
- Softmax — стандарт для розподілу ймовірностей (6 класів)
- Sigmoid на кожен клас + нормалізація
- Entmax (розріджений аналог softmax) — дає більш впевнені передбачення

**Оптимізатори:**
- AdamW з weight decay — найпоширеніший
- Adan — новіший оптимізатор, деякі учасники повідомляли про кращу збіжність

---

## Навчання

**Двоетапне навчання:**
- Stage 1 — всі дані з зваженим loss (`weight = min(n_votes/20, 1.0)`); шумні мітки з малою кількістю голосів внесені з низькою вагою
- Stage 2 — тільки семпли з `n_votes ≥ 10`; надійні мітки, ваги не потрібні

**Стратегія семплування:**
- Відбір одного sub-id на eeg_id за епоху — запобігає домінуванню записів з багатьма вікнами; кожна епоха бачить різні вікна одного запису

**Валідація:**
- StratifiedGroupKFold (5 фолдів) — стратифікація по класу + групування по patient_id щоб виключити витік між train і val
- Крос-валідація по всіх фолдах

**Ансамблі:**
- Усереднення передбачень кількох фолдів одної моделі
- Усереднення моделей з різними бекбонами або препроцесингом
- Деякі топ-рішення комбінували EEG-модель і spectrogram-модель

# Реалізація

## Препроцесинг семплу (dataset.py)

Один семпл = один `eeg_id`. Пайплайн запускається в `EEGDataset.__getitem__`:

**1. Завантаження вікна**
```
parquet → читаємо тільки 19 потрібних колонок (без EKG)
start = eeg_label_offset_seconds × 200
chunk = raw[start : start + 10 000]   # рівно 50 сек
```
Якщо вікно коротше (край файлу) — pad нулями справа.

**2. Double Banana Montage**
```
19 монополярних електродів → 16 биполярних каналів
sig[i] = electrode_A - electrode_B
Результат: (16, 10 000), float32
```
4 ланцюги по 4 канали: LL, RL, LP, RP.

**3. Очищення і масштабування**
```
nan_to_num(nan=0, posinf=1024, neginf=-1024)
clip(−1024, +1024)
÷ 32  →  діапазон ≈ [−32, +32]
```

**4. Bandpass-фільтр**
```
Butterworth 5-го порядку, SOS, 0.5–20 Гц
застосовується до всіх 16 каналів одразу (axis=-1)
```

**5. Формування 2D-зображення — 3 масштаби**

З (16, 10 000) вирізається 3 кропи навколо `center` (середина або рандом при аугментації):

| Канал | Довжина | Реальний час |
|-------|---------|--------------|
| R (0) | 2 000 семплів | 10 сек |
| G (1) | 5 000 семплів | 25 сек |
| B (2) | 10 000 семплів | 50 сек |

Кожен кроп reshape → resize до `(160, 1000)`:
```
(16, T) → reshape(16, T//10, 10) → transpose(0,2,1) → reshape(160, T//10)
         → cv2.resize → (160, 1000)
```
Висота 160 = 16 каналів × 10 мікро-семплів (50 мс) складених вертикально.

Після стекування: `(3, 160, 1000)` → z-score нормалізація по всьому тензору.

**6. Мітка і вага**
```
label  = mean(vote_cols по всіх sub-id цього eeg_id) + 0.005 (smoothing), нормалізовано → (6,)
weight = min(n_votes / 20.0, 1.0)  →  скаляр для зважування loss
```

---

## Аугментація (тільки train)

- **L-R flip** з p=0.5 — LL↔RL (канали 0–3 ↔ 4–7), LP↔RP (8–11 ↔ 12–15)
- **Random crop center** — `center = randint(2500, 7500)` замість фіксованих 5000; всі 3 кропи зсуваються разом

---

## Архітектура моделі (model.py)

```
EEGNet
├── backbone: EfficientNet-B5 (pretrained ImageNet, timm)
│    └── повертає feature map (B, 2048, H', W')
├── AdaptiveAvgPool2d(1)  →  (B, 2048)
├── Dropout(0.5)
└── Linear(2048 → 6)  +  log_softmax
```

Бекбон завантажується через `timm.create_model` з `global_pool=''` (без вбудованого пулінгу) — пулінг робимо окремо щоб мати контроль над dropout перед головою.

## Навчання (train.py)

**Функція втрат:** KL-divergence
```python
loss = KLDiv(log_probs, targets, reduction='none').sum(dim=1)  # per-sample
loss = (loss * weights).mean()   # зважене середнє по батчу (stage 1)
loss = loss.mean()               # рівномірне (stage 2)
```

**Оптимізатор:** AdamW з диференційованим LR
```
backbone params:  lr / 10   (повільніше — pretrained ваги)
голова (pool+fc): lr        (швидше — нові ваги)
weight_decay = 1e-2
```

**Scheduler:** CosineAnnealingLR на кожен stage окремо.

**Двоетапне навчання:**

| | Stage 1 | Stage 2 |
|---|---|---|
| Дані | всі eeg_id (min_votes=0) | тільки n_votes ≥ 10 |
| Loss | зважений (weight=n_votes/20) | рівномірний |
| LR | 1e-3 | 1e-4 |
| Epochs | 15 (+ early stop) | 15 (+ early stop) |

Між стейджами — відновлення найкращого чекпоінту Stage 1 перед дотренуванням.

**Валідація:**
- StratifiedGroupKFold, 5 фолдів, seed=42
- Стратифікація по `expert_consensus`, групування по `patient_id`
- Val завжди на `n_votes ≥ 10` (аргумент `--val_min_votes`)
- Early stopping: patience=5 епох без покращення val KL-loss

**Чекпоінти:** версіоновані директорії `checkpoints/modelname_vN/`, зберігається тільки найкращий по val_loss (попередній видаляється).


# Результати

Навчання загалом тривало 31 епоху: перші 13 епох — stage 1, наступні 18 — stage 2. На stage 2 використовували найкращу за val_loss модель. У підсумку вдалося досягти val_loss = 0.3037 — це добрий результат з огляду на складність задачі, відсутність ансамблю та навчання на кількох фолдах з різними сідами. Цей прогрес зроблено за два тижні продуктивної роботи; до цього найкращі моделі на тестовому наборі давали метрики близько 0.59. Усі метрики можна переглянути у файлі result/metrics.csv, а для інференсу — у ноутбуку inference_local.ipynb.
